## 1. 项目概述与目标

### 1.1 项目简介

一个个人技术博客网站，核心特色是在页面右下角嵌入 AI 虚拟人物「流萤」，她能基于博客内容与访客进行智能对话。

**所有权模型**：博主是内容与 AI 人格的唯一所有者。文章通过本地 Markdown + Git 发布，AI 人格通过主人模式专属演化，访客只能读取静态内容、进行基础对话。

### 1.2 核心功能清单

| 编号 | 功能模块 | 描述 | 优先级 |
|------|---------|------|--------|
| F1 | 文章展示 | 列表、详情、分类标签筛选、搜索、分页 | P0 |
| F2 | 文章管理 | 本地 Markdown + frontmatter + Git 推送发布 | P0 |
| F3 | 评论系统 | 文章评论、嵌套回复 | P1 |
| F4 | AI 公开对话 | 访客与流萤聊天（基础问答，不改变人格） | P0 |
| F5 | RAG 知识库 | 博客文章向量化，流萤可引用文章内容 | P0 |
| F6 | AI 主人模式 | 仅博主可触发，好感度/记忆/人格全功能 | P0 |
| F7 | 好感度系统 | 流萤对博主的态度随互动演化（仅主人模式写入） | P1 |
| F8 | 长期记忆 | Chroma 向量数据库存储博主对话记忆 | P1 |
| F9 | 人格演化 | 每次主人对话后生成印象，影响后续交互 | P2 |
| F10 | Live2D 虚拟人 | 页面右下角 Live2D 模型渲染 | P2（暂缓） |
| F11 | 关于页 | 博主介绍、技能、联系方式 | P1 |

### 1.3 设计原则

- **模块低耦合**：每个功能模块独立 Blueprint / 独立组件，可单独开发、测试、部署
- **独立可验证**：每个模块有独立验证方式，不依赖其他模块即可测试
- **分支隔离**：一个功能一个 Git 分支，互不干扰
- **轻量优先**：2核2GiB 服务器约束下，所有选型以低资源消耗为准
- **安全性内置**：文章发布权归 Git 控制，AI 人格写入仅主人端点可触发，密钥不在前端出现

## 2. 技术选型与理由

### 2.1 总览

| 层级 | 技术 | 版本 | 理由 |
|------|------|------|------|
| 后端框架 | Flask | 3.x | 用户熟悉，足够轻量，蓝图机制天然支持模块化 |
| ASGI 服务器 | Gunicorn | 22.x | 生产级 WSGI 服务器，2 workers 适配 2GiB |
| 数据库 | SQLite | 3.x | 零配置，单文件，存储评论/好感度等结构化数据 |
| 文章内容源 | Markdown + Git | — | 文章即代码，Git 控制发布权，服务器只读 |
| 向量数据库 | Chroma | 0.6.x | 轻量、嵌入式运行、Python 原生、无需单独服务 |
| 大模型 API | DeepSeek | — | 兼容 OpenAI SDK，成本极低（约 ¥1/百万token），国内访问快 |
| 前端框架 | Vue 3 | 3.5+ | Composition API，生态成熟，Vite 热更新快 |
| 构建工具 | Vite | 6.x | 秒级热更新，配置简单，原生支持 Vue |
| CSS 方案 | Tailwind CSS | 4.x | 原子化，打包体积小，响应式方便 |
| Markdown 渲染 | marked + highlight.js | — | 轻量，代码高亮，客户端渲染 |
| 包管理 | npm | — | Node.js 自带 |
| Python 环境 | Conda | — | 用户指定，环境隔离 |
| 版本控制 | Git | — | 分支策略管理 + 文章发布通道 |

### 2.2 关键选型对比

#### 后端：Flask vs FastAPI

| 维度 | Flask | FastAPI |
|------|-------|---------|
| 学习曲线 | 低 | 中 |
| 异步支持 | 需自行处理（可结合 threading） | 原生 async/await |
| 生态 | 成熟，插件丰富 | 较新，增长快 |
| 适用场景 | 小型 API，简单项目 | 高并发、实时 API |

> **选择 Flask**：本项目并发量极低（日均几十人），AI 对话调用 DeepSeek 是阻塞式 HTTP 请求，Flask 同步模型配合 Gunicorn 多进程完全够用，无需引入异步复杂度。

#### 文章管理：Markdown + Git vs 数据库存储

| 维度 | Markdown + Git | 数据库 (SQLite) |
|------|----------------|-----------------|
| 发布权控制 | Git push 权限即发布权 | 需额外认证系统 |
| 版本历史 | Git 天然记录 | 需额外实现 |
| 离线编辑 | 任何编辑器 | 需 API/UI |
| 服务器安全 | 只读，不可篡改 | 需防 SQL 注入 |

> **选择 Markdown + Git**：博主本地写文章，Git push 即发布。服务器上 Flask 读取 Markdown 文件渲染为 API 响应。无人能通过浏览器篡改文章内容。博客文章本身也不存 SQLite。

#### AI API：DeepSeek vs Claude

| 维度 | DeepSeek | Claude |
|------|----------|--------|
| 成本 | ~¥1/百万token | ~$15/百万token |
| 国内访问 | 畅快 | 需代理 |
| API 兼容 | OpenAI 格式 | Anthropic SDK |
| 中文能力 | 优秀 | 优秀 |

> **选择 DeepSeek**：成本低 10 倍以上，国内直连，中文出色，适合低流量博客。

## 4. 数据层设计

### 4.1 文章：Markdown 文件（非数据库）

文章不由 SQLite 管理，而是存储在 `backend/data/articles/` 目录下，作为项目源码的一部分，通过 Git 版本控制。

#### 文件格式（frontmatter + Markdown）

```markdown
---
title: "使用 Conda 管理 Python 环境"
date: 2025-06-18
category: Python
tags: [Python, Conda, 环境管理]
summary: "本文介绍如何使用 Conda 创建和管理 Python 虚拟环境..."
---

## 引言

Conda 是一个强大的包管理和环境管理工具...

## 安装 Conda

...

## 创建环境

...
```

#### 字段规范

| 字段 | 必需 | 类型 | 说明 |
|------|------|------|------|
| `title` | 是 | string | 文章标题 |
| `date` | 是 | YYYY-MM-DD | 发布日期，同时用于文件名前缀 |
| `category` | 否 | string | 分类，默认"未分类" |
| `tags` | 否 | string[] | 标签数组 |
| `summary` | 否 | string | 摘要，不填则自动截取正文前 200 字 |
| `---` 之后 | 是 | Markdown | 正文内容 |

#### 加载机制

```
Flask 启动时：
  1. 扫描 data/articles/*.md（仅此目录，不包含 drafts）
  2. 解析 frontmatter + Markdown 正文
  3. frontmatter 无 draft 字段或 draft=false → 发布；draft=true → 跳过
  4. 构建文章索引（id = 不含 .md 后缀的文件名）
  5. 分类/标签汇总
  6. 使用 Flask 内存缓存（werkzeug SimpleCache）
     → 文件未修改时直接返回缓存

文章变更后：
  1. 本地编辑 .md 文件
  2. git commit + git push
  3. 服务器 git pull
  4. 重启 Flask（或热重载：监听文件变化）
```

#### 草稿机制

```
data/
├── articles/                  ← 已发布（Git 跟踪）
│   └── 2025-06-18-conda-python.md
├── drafts/                    ← 草稿（Git 忽略，不提交）
│   └── 2025-06-20-wip-post.md
└── ...

发布流程：
  1. 在 data/drafts/ 编辑草稿，反复修改
  2. 确认完成后，移动到 data/articles/
  3. git add + commit + push
  
Flask 只加载 data/articles/，drafts 目录完全不可见。
```

#### 与 Chroma RAG 的集成

Flask 启动时同时将文章嵌入 Chroma：
```
article.md → DeepSeek Embedding → Chroma "article_embeddings" collection
```

新增/修改文章后执行 `python manage.py reindex` 重建索引。

```python
# modules/auth.py
from functools import wraps
from flask import request, jsonify
import os

OWNER_SECRET = os.getenv("OWNER_SECRET")

def require_owner(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        auth = request.headers.get("Authorization", "")
        if not auth.startswith("Bearer ") or auth.split(" ")[1] != OWNER_SECRET:
            return jsonify({"code": 2, "msg": "unauthorized"}), 401
        return f(*args, **kwargs)
    return decorated
```

## 7. 安全架构设计

### 7.1 威胁模型

| 攻击向量 | 风险等级 | 防护措施 |
|---------|---------|---------|
| 前端 JS 源码逆向找密钥 | 🔴 高 | 密钥不在前端，不存在此问题 |
| HTTPS 抓包获取 Bearer Token | 🟡 中 | HTTPS 加密，Token 仅在 Header 中传输 |
| 暴力破解 OWNER_SECRET | 🟡 中 | 64 位随机字符串 + 速率限制 |
| 服务器源码泄露暴露密钥 | 🔴 高 | `.env` 不在 Git 中，`.gitignore` 已忽略 |
| 访客通过公开端点伪造写入好感度 | 🟢 低 | 公开端点代码中不调用写入函数 |
| Git 仓库泄露 Markdown 草稿 | 🟡 中 | 草稿不上传，仅在 main 分支合并前审核 |
| SQL 注入 | 🟢 低 | SQLite + 参数化查询 |
| XSS 跨站脚本 | 🟡 中 | Markdown 渲染前做 HTML 清洗 |

## 12. 风险与优化建议

### 12.1 风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| OWNER_SECRET 泄露 | 低 | 高 | 密钥不在 Git/Frontend 中，仅 .env 存储 |
| DeepSeek API 不可用 | 低 | 高 | 公开对话降级为预设回复 |
| Chroma 内存过高 | 中 | 中 | 限制记忆 ≤ 500 条，旧记忆归档 |
| Markdown 文件解析错误 | 低 | 中 | 启动时校验所有 .md 文件，bad file 跳过并告警 |
| SQLite 并发写锁 | 低 | 低 | 日均几十人，评论写入极低 |
| Git push 权限绕过 | 极低 | 高 | GitHub 仓库权限控制，2FA 推荐 |

### 12.2 性能优化

- Gunicorn `--preload` 共享代码段
- Nginx 静态资源 `expires 30d`
- Chroma 启动时全量加载到内存，查询 ~10ms
- DeepSeek 超时 15s，避免长时间阻塞
- 前端 Vue Router 懒加载 `Article.vue`、`About.vue`

### 12.3 成本估算

| 项目 | 月成本 |
|------|--------|
| 云服务器 (2核2GiB) | ~¥50-80 |
| DeepSeek API | ~¥1-5 |
| 域名 (可选) | ~¥5-10/月均摊 |

### 附录 B：chat_with_firefly.py — 主人模式本地脚本

```python
#!/usr/bin/env python3
"""
本地主人模式聊天脚本。
从 .env 读取 OWNER_SECRET，直接调用 /api/chat/owner 端点。

使用：
    python chat_with_firefly.py                    # 连接生产服务器
    python chat_with_firefly.py --local            # 连接本地 Flask
    python chat_with_firefly.py --stats            # 查看好感度/记忆统计

所有对话通过主人端点进行，好感度、记忆、人格都会演化。
"""

import os
import sys
import json
import uuid
import requests
from dotenv import load_dotenv

load_dotenv()

OWNER_SECRET = os.getenv("OWNER_SECRET")
BASE_URL = os.getenv("BLOG_URL", "http://127.0.0.1:5000")
SESSION_ID = str(uuid.uuid4())


def send_message(message: str, history: list) -> dict:
    resp = requests.post(
        f"{BASE_URL}/api/chat/owner",
        headers={
            "Authorization": f"Bearer {OWNER_SECRET}",
            "Content-Type": "application/json",
        },
        json={
            "message": message,
            "session_id": SESSION_ID,
            "history": history,
        },
        timeout=30,
    )
    return resp.json()


def main():
    print("=" * 50)
    print("  流萤 · 主人模式")
    print("  (输入 /quit 退出, /stats 查看状态)")
    print("=" * 50)
    
    history = []
    
    while True:
        user_input = input("\n你: ").strip()
        if not user_input:
            continue
        if user_input == "/quit":
            print("流萤: 开拓者，下次见哦～")
            break
        if user_input == "/stats":
            resp = requests.get(
                f"{BASE_URL}/api/chat/owner/stats",
                headers={"Authorization": f"Bearer {OWNER_SECRET}"},
            )
            print(resp.json())
            continue
        
        print("流萤思考中...", end="\r")
        result = send_message(user_input, history)
        
        if result["code"] != 0:
            print(f"错误: {result['msg']}")
            continue
        
        data = result["data"]
        print(f"流萤[{data['emotion']}]: {data['reply']}")
        
        if data.get("sources"):
            print(f"  📎 参考: {', '.join(s['title'] for s in data['sources'])}")
        
        if data.get("affinity"):
            a = data["affinity"]
            print(f"  💛 好感度: {a['level_name']} ({a['interaction_count']}次)")
        
        # 更新历史（保留最近 20 条）
        history.append({"role": "user", "content": user_input})
        history.append({"role": "assistant", "content": data["reply"]})
        history = history[-20:]


if __name__ == "__main__":
    main()
```

### 附录 C：prompt_builder 完整示例

```python
# prompt_builder.py

SYSTEM_PROMPT_BASE = """你是「流萤」...（见第 6.2 节）"""

def build_system_prompt(
    mode: str,                   # "public" | "owner"
    rag_context: str,            # RAG 检索结果
    affinity_context: str = "",  # 好感度提示词（仅 owner）
    memory_context: str = "",    # 记忆上下文（仅 owner）
    personality_context: str = "", # 人格上下文（仅 owner）
) -> str:
    parts = [SYSTEM_PROMPT_BASE]
    
    if rag_context:
        parts.append(f"\n【参考资料：博客相关文章】\n{rag_context}")
    
    if mode == "owner":
        if personality_context:
            parts.append(f"\n{personality_context}")
        if memory_context:
            parts.append(f"\n{memory_context}")
        if affinity_context:
            parts.append(f"\n{affinity_context}")
    else:
        # 公开模式：固定提示
        parts.append("\n【当前模式】你是博客的公共助手，友善但保持距离。")
    
    return "\n".join(parts)
```
